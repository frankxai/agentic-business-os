# /weekly-review — close the week, update memory

**Usage:** `/weekly-review` — run Friday or Sunday.

## Pipeline

1. Git log the week: what actually shipped vs. the `/weekly-content` plan
2. Ask the operator 3 questions, one at a time: What moved? What stalled and why? What did you learn about the customer/market this week?
3. Write the review to `docs/intelligence/weekly/<year>-W<week>-review.md`
4. Update `docs/intelligence/MEMORY.md` — add/update index lines for anything durable (a market insight, a supplier fact, a decision made informally)
5. If a real decision was made this week, formalize it: `docs/intelligence/decisions/<next-number>-<slug>.md`

The review is 15 minutes, not an hour. The value is the compounding memory, not the ceremony.
