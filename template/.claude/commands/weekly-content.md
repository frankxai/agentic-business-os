# /weekly-content — plan the week's content

**Usage:** `/weekly-content` — run Sunday or Monday.

## Pipeline

1. Load `docs/intelligence/MEMORY.md` + last entry in `docs/intelligence/weekly/`
2. Review what shipped last week (git log on `content/`)
3. Propose the week: **1 journal post + 1 social distribution day + 1 product/ops task**, each with a one-line rationale tied to the current business stage (pre-launch = build the notify list + the research library)
4. Write the plan to `docs/intelligence/weekly/<year>-W<week>-plan.md`
5. Keep it executable by one person with a day job. Three items maximum. A plan that doesn't fit the founders' real week is theatre.
