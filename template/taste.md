## The restraint test

Before adding anything to a screen:
1. Does removing this make the screen worse? If no, remove it.
2. Does this exist already in `components/`? Use it or extend it.
3. Does this carry meaning? Decoration without meaning is noise.
4. Does this raise the page's signal-to-noise ratio? If no, cut it.

The goal: **everything on the page has a job**, nameable in one sentence.

## The polish pass

Before any visual ships:
1. **Squint test** — hierarchy readable when blurred?
2. **Mobile test** — headline above fold, CTA thumb-reachable?
3. **Contrast test** — body text survives minimum brightness?
4. **Keyboard test** — focus rings visible, tab order logical?
5. **Reading test** — copy read aloud sounds like a person?
6. **Claims test** — `@claims-guard` on every copy change. Non-negotiable.
7. **Compress test** — remove the second-most-prominent element; better without it? Remove for real.
